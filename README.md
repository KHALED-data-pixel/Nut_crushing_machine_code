# Nut Processing & Sorting Machine — Vision Pipeline

Machine learning vision system for classifying and sizing **argan**, **almond**, and **peanut** nuts.  
Detects nut type and dimensions via camera, then sends a calibrated cracking force command to the cracking station over serial.

---

## Quick Start

### 1. Install Python Dependencies

```bash
pip install -r requirements.txt
```

> **Note:** If your PC has no NVIDIA GPU, replace `onnxruntime-gpu` with `onnxruntime` in requirements.txt.

---

### 2. Edit `config.json`

Open `config.json` and set:

| Key | Description | Example |
|-----|-------------|---------|
| `camera_index` | USB camera index (0 = first camera) | `0` |
| `serial_port` | Arduino COM port (Windows) | `"COM3"` |
| `serial_baud` | Baud rate (must match firmware) | `115200` |
| `mm_per_px` | Metric calibration (see Step 4 below) | `0.3125` |
| `confidence_threshold` | Minimum detection confidence | `0.6` |

---

### 3. Create Dataset Folder Structure

```bash
python dataset_yaml_generator.py
```

This creates all required `dataset/` subdirectories and `dataset.yaml`.

---

### 4. Camera Calibration

#### 4a. Print a 9×6 chessboard
- Print `calibration_chessboard.pdf` (or generate at [calib.io](https://calib.io/pages/camera-calibration-pattern-generator)).
- Square size: **10 mm** (physical, measured with calipers).
- Attach to a rigid flat surface.

#### 4b. Capture chessboard images

```bash
python capture_samples.py --class calib --count 30
```

Move the chessboard to **30 different positions and angles** while pressing SPACE to capture.  
Images saved to `dataset/images/raw/calib/`.

#### 4c. Compute calibration

```bash
python calibrate_camera.py
```

Output: `calib.npz`. Re-projection error should be **< 1.0 px**.

#### 4d. Set `mm_per_px` (metric calibration)

1. Place a known-size reference object at the inspection plane (e.g., a **30 mm coin**).
2. Open the live camera: `python capture_samples.py --class calib --count 1`
3. Measure the coin diameter in pixels using `cv2.selectROI` or any image editor.
4. Calculate: `mm_per_px = real_width_mm / pixel_diameter`
5. Update `config.json`.

---

### 5. Collect Training Images

For each nut class, place nuts one at a time on the inspection rail:

```bash
python capture_samples.py --class argan  --count 200
python capture_samples.py --class almond --count 200
python capture_samples.py --class peanut --count 200
```

Images go to `dataset/images/raw/<class>/`.

---

### 6. Annotate Images

Use **[labelImg](https://github.com/HumanSignal/labelImg)** (free, offline):

```bash
pip install labelImg
labelImg dataset/images/raw/argan  dataset/dataset.yaml
```

- Set format to **YOLO**.
- Draw a bounding box around each nut → labelImg saves a `.txt` label file.
- Split annotated images 80/20 into `dataset/images/train/` and `dataset/images/val/`.
- Place corresponding `.txt` labels in `dataset/labels/train/` and `dataset/labels/val/`.

**Label format** (one line per nut, normalised 0-1):
```
class_id  cx  cy  width  height
0 0.512 0.487 0.134 0.180
```

Class IDs:
- `0` = argan
- `1` = almond
- `2` = peanut

---

### 7. Train the Model

```bash
# GPU (recommended)
python train_yolo.py --epochs 100 --batch 16 --device 0

# CPU only
python train_yolo.py --epochs 100 --batch 4 --device cpu
```

Training progress is shown in the terminal. When complete, `best.onnx` is copied to the project root.

---

### 8. Upload Arduino Firmware

1. Open `cracking_station_firmware/cracking_station_firmware.ino` in Arduino IDE.
2. Edit the pin constants (`STEP_PIN`, `DIR_PIN`, etc.) to match your wiring.
3. **Calibrate** `STEPS_PER_NEWTON` using a force gauge on the physical machine.
4. Select your board and port → **Upload**.
5. Open Serial Monitor at **115200 baud** — you should see `READY`.

---

### 9. Run Live Inference

```bash
# Full system (camera + serial to Arduino)
python infer_nut.py

# Testing without machine (no serial)
python infer_nut.py --no-serial

# Headless (no display window, logs only)
python infer_nut.py --no-display
```

**Controls:** Press `Q` or `ESC` in the display window to stop.

## Force Lookup Table

Edit `force_table.py` to adjust cracking forces after physical calibration:

| Nut | Small | Medium | Large |
|-----|-------|--------|-------|
| Argan | 400 N (< 13 mm) | 500 N (13–16 mm) | 600 N (> 16 mm) |
| Almond | 250 N (< 18 mm) | 300 N (18–23 mm) | 380 N (> 23 mm) |
| Peanut | 80 N (< 10 mm) | 100 N (10–13 mm) | 140 N (> 13 mm) |

Test the table independently:
```bash
python force_table.py
```

## Serial Protocol (Python ↔ Arduino)

| Direction | Message | Meaning |
|-----------|---------|---------|
| PC → Arduino | `FORCE=450\n` | Apply 450 N cracking force |
| PC → Arduino | `HOME\n` | Return jaw to home position |
| PC → Arduino | `STATUS\n` | Query current state |
| Arduino → PC | `ACK=450\n` | Command acknowledged |
| Arduino → PC | `READY\n` | Startup confirmation |
| Arduino → PC | `ERR=<msg>\n` | Error message |

## Troubleshooting

| Problem | Solution |
|---------|----------|
| `Cannot open camera index 0` | Try `--config` and change `camera_index` to `1` or `2` |
| `ONNX model not found: best.onnx` | Run `train_yolo.py` first |
| `Calibration file not found: calib.npz` | Run `calibrate_camera.py` first |
| `Cannot open serial port COM3` | Check Device Manager → COM port number; change in `config.json` |
| Low confidence detections | Reduce `confidence_threshold` to `0.4`, or collect more training data |
| Wrong force values | Recalibrate `mm_per_px` and `STEPS_PER_NEWTON` |
| RMS re-projection error > 2.0 px | Capture more chessboard images with more variety of angles |

## Dataset Options (Fastest to Slowest)

1. **Synthetic from SolidWorks** ← Recommended (no physical machine needed)  
   Render nuts in PhotoView 360 from the camera viewpoint. Extract bounding boxes from 3D coordinates via SolidWorks API macro.

2. **Public datasets**  
   Download from [Roboflow Universe](https://universe.roboflow.com/) — search "almond", "peanut". Argan is rare; manual collection needed.

3. **Real images** (requires physical machine)  
   Use `capture_samples.py` once the machine is built.
